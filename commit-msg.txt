fix(tilesetview): refresh tile layout on tileset image resize (fixes #3923)

When replacing a tileset source image with a taller version (additional rows),
the Tile Animation Editor's tileset view did not show the new bottom tiles
until the tileset document was closed and reopened, even though the main
tileset view updated correctly.

This commit fixes the issue by ensuring the TilesetView properly reacts to
tileset changes:

- Made TilesetView listen to TilesetDocument::tilesetChanged(Tileset*)
  and, when its own tileset changes, call scheduleDelayedItemsLayout() and
  refreshColumnCount() to force correct relayout and update the visible tile grid.

- Simplified TileAnimationEditor::tilesetChanged() to:
  - Reset the current tile
  - Update the tileset view background
  - Forward tilesetChanged() to its TilesetModel
  - Rely on TilesetView's new handling for layout updates (no redundant view refresh needed here)

Tested by:
- Creating a tileset from tiles_base.png (8 rows)
- Replacing it with tiles_taller.png (14 rows)
- Verifying new bottom tiles appear immediately in Tile Animation Editor
  without closing/reopening the dialog

No regression in main tileset view refresh or animation frame selection.
